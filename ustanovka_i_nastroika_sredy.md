# Подготовка программной среды для работы с TurtleBot3

## Общая информация

Для дальнейшей работы с виртуальной моделью `TurtleBot3` требуется подготовить программную среду, в которой будут установлены операционная система `Ubuntu`, необходимые инструменты разработки и программные компоненты для последующей установки `ROS` и пакетов робота. В рамках данного этапа в качестве среды выполнения была выбрана виртуальная машина, создаваемая в `VirtualBox`.

Использование виртуальной машины позволяет развернуть отдельную рабочую среду `Ubuntu` без изменения основной операционной системы компьютера. Такой подход удобен для учебных целей, поскольку дает возможность безопасно выполнять установку и настройку необходимых компонентов.

## Выбор среды развертывания

Для подготовки программной среды было принято решение использовать:

- `VirtualBox` как средство виртуализации;
- `Ubuntu 22.04 LTS` как основную операционную систему виртуальной машины.

Выбор `Ubuntu 22.04 LTS` обусловлен тем, что данная версия широко используется для работы с `ROS` и подходит для последующего изучения симуляции и навигации робота `TurtleBot3`.

## Загрузка необходимых компонентов

Перед началом установки необходимо загрузить следующие компоненты:

- установочный файл `VirtualBox` для Windows;
- образ операционной системы `Ubuntu 22.04 LTS` в формате `ISO`.

Для загрузки `VirtualBox` на официальной странице необходимо выбрать вариант:

- `Windows hosts`

Именно этот пункт предназначен для установки `VirtualBox` на компьютер с операционной системой Windows.

## Установка VirtualBox

Установка `VirtualBox` выполняется стандартным способом:

1. Запустить скачанный установочный файл.
2. Подтвердить установку программы.
3. Оставить параметры установки по умолчанию.
4. Дождаться завершения установки.
5. При необходимости перезагрузить компьютер.

На данном этапе установка дополнительных компонентов, таких как `Extension Pack`, не является обязательной, поскольку для базового развертывания виртуальной машины и установки `Ubuntu` достаточно основного пакета `VirtualBox`.

## Создание виртуальной машины

После установки `VirtualBox` необходимо создать новую виртуальную машину для `Ubuntu`.

При создании виртуальной машины следует выполнить следующие действия:

1. Открыть `VirtualBox`.
2. Нажать кнопку создания новой виртуальной машины.
3. Указать имя виртуальной машины, например `Ubuntu 22.04`.
4. Выбрать тип системы `Linux`.
5. Выбрать версию `Ubuntu (64-bit)`.
6. Подключить ранее скачанный `ISO-образ Ubuntu`.

## Рекомендуемые параметры виртуальной машины

Для комфортной работы с системой рекомендуется задать следующие параметры:

- оперативная память: не менее `8 ГБ`;
- количество ядер процессора: `4`;
- объем виртуального диска: не менее `40 ГБ`;
- тип диска: динамически расширяемый;
- видеопамять: максимально доступное значение;
- включение `3D Acceleration`, если функция доступна.

Выделение достаточного объема ресурсов особенно важно, если в дальнейшем планируется запуск графических приложений и симуляции робота.

## Установка Ubuntu 22.04 LTS

После настройки виртуальной машины выполняется установка операционной системы `Ubuntu`.

Основная последовательность действий:

1. Запустить созданную виртуальную машину.
2. Дождаться загрузки установщика `Ubuntu`.
3. Выбрать пункт `Install Ubuntu`.
4. Указать язык установки.
5. Выбрать раскладку клавиатуры.
6. Выбрать стандартный вариант установки.
7. На этапе разметки диска подтвердить использование виртуального диска.
8. Указать часовой пояс.
9. Создать имя пользователя и пароль.
10. Дождаться завершения установки.
11. Перезагрузить виртуальную машину.

Важно учитывать, что на этапе выбора диска изменения вносятся только во внутренний диск виртуальной машины и не затрагивают основную операционную систему пользователя.

## Первичная настройка после установки

После первого запуска `Ubuntu` необходимо выполнить начальную настройку системы. Для этого рекомендуется открыть терминал и выполнить обновление пакетов:

```bash
sudo apt update
sudo apt upgrade -y
```

Эти команды позволяют обновить список доступных пакетов и установить актуальные версии системных компонентов.

После завершения обновления рекомендуется выполнить перезагрузку виртуальной машины, чтобы система начала работу в актуальном состоянии:

```bash
sudo reboot
```

## Дополнительные действия после установки

После успешной установки `Ubuntu 22.04 LTS` и первичного обновления системы были выполнены дополнительные действия, повышающие удобство и надежность дальнейшей работы.

### Создание снимка виртуальной машины

Перед переходом к установке `ROS` был создан снимок виртуальной машины (`snapshot`). Это позволяет сохранить текущее рабочее состояние `Ubuntu` и при необходимости быстро вернуться к нему в случае ошибок при дальнейшей настройке среды.

### Настройка общего буфера обмена

Для упрощения работы с командами в `VirtualBox` был включен двунаправленный общий буфер обмена. Это позволяет копировать команды из основной системы Windows и вставлять их в терминал `Ubuntu`, что особенно важно при выполнении длинных команд установки.

Для этого в окне виртуальной машины используется следующий путь:

- `Устройства -> Общий буфер обмена -> Двунаправленный`

При необходимости также можно включить двунаправленное перетаскивание:

- `Устройства -> Перетаскивание -> Двунаправленное`

Вставка текста в терминал `Ubuntu` выполняется сочетанием клавиш:

```text
Ctrl + Shift + V
```

## Подготовка к установке ROS 2 Humble

После завершения настройки базовой системы выполняется установка `ROS 2 Humble`, которая будет использоваться в дальнейшем для работы с `TurtleBot3`.

Последовательность команд для установки выглядит следующим образом:

```bash
sudo apt install -y software-properties-common curl gnupg lsb-release
sudo add-apt-repository universe
sudo curl -sSL https://raw.githubusercontent.com/ros/rosdistro/master/ros.key -o /usr/share/keyrings/ros-archive-keyring.gpg
echo "deb [arch=$(dpkg --print-architecture) signed-by=/usr/share/keyrings/ros-archive-keyring.gpg] http://packages.ros.org/ros2/ubuntu $(. /etc/os-release && echo $UBUNTU_CODENAME) main" | sudo tee /etc/apt/sources.list.d/ros2.list > /dev/null
sudo apt update
sudo apt install -y ros-humble-desktop python3-colcon-common-extensions python3-rosdep python3-vcstool
sudo rosdep init
rosdep update
echo "source /opt/ros/humble/setup.bash" >> ~/.bashrc
source ~/.bashrc
```

После установки рекомендуется проверить, что `ROS 2` подключен корректно:

```bash
printenv | grep ROS
ros2 --version
```

На практике проверка была выполнена через вызов справки по одной из команд `ROS 2`:

```bash
ros2 topic --help
```

Успешный вывод списка доступных подкоманд подтверждает корректную установку и подключение `ROS 2 Humble`.

## Статус пункта Install ROS 2 on Remote PC

В документации TurtleBot3 присутствует этап `Install ROS 2 on Remote PC`. В рамках текущей работы роль `Remote PC` выполняет созданная виртуальная машина с `Ubuntu 22.04 LTS`. Следовательно, установка `ROS 2 Humble`, уже выполненная в данной виртуальной машине, фактически и является выполнением этого пункта.

Это означает, что повторно устанавливать `ROS 2` не требуется. Следующим этапом становится установка дополнительных пакетов и компонентов, необходимых именно для работы с `TurtleBot3`, симуляцией, навигацией и визуализацией.

## Следующие пакеты для TurtleBot3

После установки `ROS 2 Humble` необходимо дополнительно установить связанные пакеты, используемые в среде `TurtleBot3`:

```bash
sudo apt install -y ros-humble-gazebo-* ros-humble-cartographer ros-humble-cartographer-ros ros-humble-navigation2 ros-humble-nav2-bringup git
```

Факт установки указанных компонентов был дополнительно проверен командой:

```bash
dpkg -l | grep -E "ros-humble-(gazebo|cartographer|navigation2|nav2-bringup)|git"
```

По результатам проверки в системе присутствуют пакеты `ros-humble-cartographer`, `ros-humble-cartographer-ros`, `ros-humble-navigation2`, `ros-humble-nav2-bringup`, набор пакетов `ros-humble-gazebo-*`, а также `git`. Это подтверждает, что базовые компоненты для симуляции, построения карт и навигации уже установлены.

Затем создается рабочее пространство и загружаются исходные пакеты `TurtleBot3`:

```bash
mkdir -p ~/turtlebot3_ws/src
cd ~/turtlebot3_ws/src
git clone -b humble https://github.com/ROBOTIS-GIT/DynamixelSDK.git
git clone -b humble https://github.com/ROBOTIS-GIT/turtlebot3_msgs.git
git clone -b humble https://github.com/ROBOTIS-GIT/turtlebot3.git
cd ~/turtlebot3_ws
colcon build --symlink-install
echo "source ~/turtlebot3_ws/install/setup.bash" >> ~/.bashrc
echo "export TURTLEBOT3_MODEL=waffle_pi" >> ~/.bashrc
source ~/.bashrc
```

Проверить настройку модели можно командой:

```bash
printenv | grep TURTLEBOT3
```

На практике в переменную окружения была успешно добавлена модель:

```bash
TURTLEBOT3_MODEL=waffle_pi
```

В ходе выполнения данного этапа была создана и настроена виртуальная машина `Ubuntu 22.04 LTS` в среде `VirtualBox`. После установки операционной системы были выполнены обновление системы, создание снимка виртуальной машины и настройка базовых параметров, необходимых для дальнейшей работы.

Затем в виртуальной машине была успешно установлена среда `ROS 2 Humble`. Корректность установки была подтверждена запуском команд `ROS 2` и получением справочной информации по доступным подкомандам.

После этого были установлены дополнительные зависимости, необходимые для работы с `TurtleBot3`, включая пакеты для симуляции, навигации и картографирования. В рабочем пространстве `~/turtlebot3_ws` были размещены исходные коды пакетов `DynamixelSDK`, `turtlebot3`, `turtlebot3_msgs` и `turtlebot3_simulations`.

Сборка рабочего пространства была выполнена успешно командой:

```bash
colcon build --symlink-install
```

По итогам сборки были получены и подключены пакеты, необходимые для дальнейшей работы с виртуальной моделью робота.

## Результат запуска виртуальной модели на текущем этапе

После подключения собранного рабочего пространства был выполнен запуск виртуальной модели `TurtleBot3 Waffle Pi` с помощью команды:

```bash
ros2 launch turtlebot3_gazebo empty_world.launch.py
```

В результате запуска было установлено, что:

- серверная часть `Gazebo` успешно запускается;
- модель робота `TurtleBot3 Waffle Pi` успешно создается в симуляции;
- пакеты `turtlebot3_gazebo` и связанные зависимости обнаруживаются системой корректно;
- ошибка возникает на стороне графического клиента `gzclient`.

Таким образом, на текущем этапе подтверждено, что программная среда установлена корректно, рабочее пространство собрано успешно, а запуск симуляционной части в целом выполняется. При этом в среде `VirtualBox` выявлено ограничение, связанное с работой графического интерфейса `Gazebo`, из-за которого происходит аварийное завершение `gzclient`.
